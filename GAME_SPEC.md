# Attack of the Dragon 仕様書 v0.1

作成日: 2026-07-01

## 1. コンセプト

縦持ちスマホ向けのエンドレス飛行アクション。

プレイヤーはぽっちゃりした子供の竜を操作し、タップで上昇、放置で下降しながら空を進む。フラッピーバードの土管の代わりに、空中の敵編隊や回転配置などの敵パターンが出現する。

敵を見つけると竜は自動で炎を吐いて攻撃する。敵はすべて撃破可能だが、配置や出現タイミングによっては攻撃が間に合わないパターンがあり、その場合は飛行操作で敵や弾をよける必要がある。

## 2. 目標体験

- 片手・1タップだけで遊べる。
- 見た目はかわいいが、避ける判断はしっかり必要。
- 攻撃操作は不要で、プレイヤーは飛行ルート選択に集中する。
- 敵を倒す爽快感と、倒しきれない敵を避ける緊張感を交互に出す。
- 1回のプレイは短く、スコア更新を狙って繰り返し遊ぶ。

## 3. 対応環境

- 画面: スマホ縦画面固定
- 操作: タッチ入力
- 想定解像度基準: 1080 x 1920
- 表示方式: 2D横スクロール
- 実装技術: Flutter
- プレイヤー位置: 画面左寄りに固定し、敵が右から左へ流れ、背景も低速で右から左へスクロールする

## 4. アート方針

- スタイル: 2D手描き
- 推奨トーン: かわいいファンタジー、少しコミカル、視認性重視
- 主人公: ぽっちゃりした子供の竜
- アニメーション: なめらか重視
- 背景: ActionMaker-Steam の既存背景素材を利用する

### 4.1 主人公デザイン仮決め

- シルエット: 丸い胴体、大きめの頭、小さめの角、短い手足、大きめの翼
- 表情: 少し必死だがかわいい
- 色: 背景と敵から分離しやすい暖色系を第一候補にする
- 読みやすさ: 小さい画面でも翼、頭、胴体が判別できること

## 5. ゲームルール

### 5.1 勝敗

- ゲーム形式: エンドレス
- 目的: できるだけ長く飛び、距離スコアを伸ばす
- ゲームオーバー条件:
  - 敵本体に接触
  - 敵の弾に接触
  - 画面下に落下
  - 画面上の天井判定に接触
- ライフ: なし。1回接触で即ゲームオーバー

### 5.2 スコア

- メインスコア: 飛行距離
- 加算方式: 最初のタップでゲーム開始した後、生存時間に応じて自動加算
- 敵撃破スコア: MVPでは任意表示。実装する場合も距離スコアより小さく扱う
- 表示:
  - プレイ中: 画面上部中央に距離スコア
  - リザルト: 今回スコア、ベストスコア、撃破数

### 5.2.1 オンラインスコアボード

- スコアは Cloudflare D1 に保存し、Cloudflare Worker 経由でオンライン共有できるようにする
- スコアボード画面は Cloudflare Worker のランキングAPIを読み込んで表示する
- オンラインランキングは最大1万位まで閲覧できるようにする
- 同梱フォールバックデータ: `assets/data/leaderboard.json`
- 保存する項目:
  - プレイヤー名
  - 距離スコア
  - 撃破数
  - 記録日時
  - ゲームバージョン
- プレイヤー名は設定画面で変更できる
- プレイヤー名はローカルに保存し、スコア投稿時に使用する
- 未設定の場合は `Dragon` を初期名として扱う
- プレイヤー名は1文字から14文字まで
- 初回起動時のプレイヤー名は `Dragon00000000` 形式で、8桁の数字部分をランダム生成する
- プレイヤー名には制御文字、改行、URL、HTMLタグを使えない
- ローカル端末には自己ベストと直近スコアを保存し、オンライン取得に失敗した場合はローカル表示にフォールバックする
- ゲームクライアントにはデータベースの認証情報を埋め込まない
- ゲームクライアントには D1 の認証情報や管理キーを埋め込まず、公開可能な Cloudflare Worker のURLだけを埋め込む
- オンラインランキングの接続先は `SCORE_SUBMIT_URL` としてアプリに埋め込む
- `SCORE_SUBMIT_URL` はスコア投稿だけでなく、オンラインランキング取得、ラン開始トークン発行、アカウント認証連携にも使用する
- 本番 Worker URL は公開されてもよい情報としてソースコードに持たせ、リリースビルド時のデフォルト値として使用する
- 通常のリリースビルドでは `--dart-define` 指定を不要にする
- デバッグビルドやテストでは未指定時の `SCORE_SUBMIT_URL` は空とし、オンライン接続したい場合や開発用、検証用の Worker を使う場合だけ `--dart-define=SCORE_SUBMIT_URL=...` で上書きする
- `SCORE_SUBMIT_URL` を明示的に空にした場合、オンライン投稿とラン開始トークン取得は行わず、同梱ランキングまたはローカルスコア表示へフォールバックする
- スコア投稿は Cloudflare Worker へ送信し、Worker が D1 を更新する
- MVPで投稿エンドポイントをまだ用意しない場合は、オンラインスコアボードは読み取り専用として始める

### 5.3 開始フロー

1. タイトル画面で `スタート` を押す
2. プレイ画面へ遷移し、竜が画面左寄りの空中に静止した状態で待機する
3. 待機中は重力、スコア加算、敵出現、自動攻撃、当たり判定を開始しない
4. プレイヤーが画面をタップするとゲーム開始
5. 開始タップは通常のタップ上昇としても扱い、竜に上方向の力を加える

この待機状態を `Ready`、実際に飛行が始まった状態を `Playing` と呼ぶ。

### 5.4 当たり判定

- 当たり判定は見た目のままにする
- プレイヤー、敵、敵弾の当たり判定を意図的に小さくしない
- 透明な余白は判定に含めない
- 実装では、スプライトの見た目に合わせた円、楕円、またはポリゴン判定を使う
- 見た目と判定がずれていると感じる場合は、絵側または判定形状を見た目に合わせて修正する
- Ready 中は当たり判定を開始しない

### 5.5 難易度

- v0.1では難易度は一定
- スクロール速度、敵HP、敵出現間隔は基本的に固定
- ランダムな敵パターン選択によって展開に変化を出す
- 将来案として、一定距離ごとに速度や危険パターン比率を上げる余地を残す

## 6. 操作仕様

### 6.1 基本操作

- Ready中のタップ: ゲームを開始し、竜に上方向の瞬間的な力を加える
- Playing中のタップ: 竜に上方向の瞬間的な力を加える
- 放置: 重力で下降
- 長押し: なし
- 左右移動: なし
- 攻撃ボタン: なし

### 6.2 飛行挙動

- 竜の横位置は固定
- スタートボタン押下後、最初のタップまでは竜の位置を固定し、落下させない
- 連続タップで高度を維持または上昇できる
- タップしすぎると天井に当たり、放置しすぎると落下する
- 操作感はフラッピーバードに近いが、敵回避のために上下の微調整が重要
- プレイヤーと敵の画面内サイズ感はフラッピーバードを参考にし、スマホ画面で常に大きく読みやすい比率にする
- 通り抜ける隙間は画面高の約24%を基準にし、簡単すぎず理不尽すぎない幅に調整する

## 7. 攻撃仕様

### 7.1 自動炎攻撃

- 敵が画面内に入ると自動で攻撃対象候補になる
- 攻撃対象: 通常はプレイヤーに最も近い敵。敵が多い場面では近い敵2体まで同時に狙う
- 射程: 画面右端まで
- 攻撃方向: 対象に向かって自動照準
- 攻撃頻度: かなり高め。敵が増えても爽快に倒せるよう、短い一定間隔で炎を連射する
- プレイヤー操作で攻撃対象は選べない

### 7.2 炎の扱い

- 炎は弾または短いビーム状エフェクトとして表現する
- 敵に命中するとダメージを与える
- 敵HPが0になると撃破
- 一度に攻撃できる敵は1体
- 貫通はしない

### 7.3 攻撃が間に合わない状況

敵はすべて撃破可能だが、次のような状況では接触前に倒しきれない。

- 敵のHPが高い
- 敵の出現位置がプレイヤーに近い
- 敵が複数体まとまって出てくる
- 近い敵にターゲットを吸われ、本当に避けたい敵への攻撃が遅れる
- 敵が回転・上下移動して接触ルートに入ってくる

この場合、プレイヤーは飛行操作で安全な隙間を抜ける。

## 8. 敵仕様

### 8.1 基本ルール

- 敵はすべて空中に出現する
- 地上敵は出さない
- 敵本体に触れると即ゲームオーバー
- 一部の敵は弾を撃つ
- 敵は単体ではなく、主にパターン単位で出現する

### 8.2 敵タイプ

v0.1では5種類を実装候補とする。

| ID | 名前 | 役割 | 動き | HP | 攻撃 | 狙い |
| --- | --- | --- | --- | --- | --- | --- |
| E01 | コウモリ | 基本敵 | 直進 | 低 | なし | 炎で倒す気持ちよさを作る |
| E02 | ツノバード | 速い敵 | 直進・やや速い | 低 | なし | 早めの上下判断を要求する |
| E03 | ぷかぷかスライム | 障害物寄り | ゆっくり上下 | 中 | なし | 倒すか避けるかを迷わせる |
| E04 | 空中メイジ | 遠距離敵 | 一定位置から流れる | 中 | 弾あり | 弾回避を入れる |
| E05 | アーマーガーゴイル | 高耐久敵 | 直進・遅い | 高 | なし | 倒しきれない配置を作る |

名前と見た目は仮。実装前に世界観に合わせて調整する。

### 8.3 敵弾

- v0.1では空中メイジのみ弾を撃つ
- 弾は左方向へ直進し、見た目の向きは飛行方向に合わせる
- 弾に触れると即ゲームオーバー
- 弾速は敵本体より少し速い
- 弾数は画面内で多くなりすぎないよう制限する

## 9. 敵パターン仕様

### 9.1 パターン出現

- 敵はランダムに選ばれたパターンとして出現する
- パターンごとに敵タイプ、数、配置、移動、回転を定義する
- 出現間隔は一定
- 敵パターンの出現は Ready 中には始めず、最初のタップで Playing になってから開始する
- 連続で理不尽なパターンが出ないよう、危険度の高いパターンにはクールダウンを持たせる

### 9.2 パターン分類

| 種別 | 説明 | プレイヤー行動 |
| --- | --- | --- |
| 撃破パターン | 炎で倒しきれる配置 | 基本ルート維持 |
| 回避パターン | 倒しきれない敵が混ざる配置 | 隙間を抜ける |
| 混合パターン | 一部を倒し、一部を避ける配置 | ターゲット順を読んで高度調整 |
| 弾幕パターン | 少数の弾を避ける配置 | 弾と敵本体の両方を見る |

| P02 | 上下ゲート | 上下に大型敵、隙間の上下に小型・中型敵が追従 | 中 |
| P03 | 回転リング | 小型敵12体がゆっくり円形に回る | 中 |
| P04 | ガーゴイル壁 | 高耐久敵がさらに高密度で縦に並び、隙間に小型敵も混ざる | 高 |
| P05 | メイジ護衛 | 空中メイジの周囲に小型敵、速い敵、中型敵を多めに配置 | 中 |

| P07 | 斜めライン | 敵9体が斜めに並び、上昇または下降ルートを要求 | 中 |
| P08 | 交互上下 | 上、下、上、下の順に敵10体が流れる | 中 |

## 10. 画面構成

### 10.1 タイトル画面

- スタート
- 設定
- スコアボード

タイトル画面には上記3項目以外のボタンや常設情報を置かない。ベストスコアはタイトル画面に直接表示せず、スコアボード内で確認できるようにする。

### 10.2 プレイ画面

- 中央上: 距離スコア
- 左寄り: プレイヤー竜
- 右から左: 敵、敵弾、低速スクロール背景
- 攻撃エフェクト: 竜から対象敵へ炎
- 炎弾と敵弾は飛行方向へ回転させ、アセットの向きが移動方向とずれないようにする
- UIは最小限にする

スタートボタン押下直後のプレイ画面は Ready 状態にする。竜は空中に静止し、画面タップで開始する。Ready 中は必要に応じて「タップしてスタート」程度の短い表示だけ出す。

### 10.3 リザルト画面

- 今回の距離スコア
- ベストスコア
- 撃破数
- リトライボタン
- タイトルへ戻るボタン

### 10.4 スコアボード画面

- 表示・構成はスイカゲームのスコアボードと同じ見せ方にする
- ベストスコア
- 過去プレイの上位スコア
- Cloudflare D1 から取得したオンラインランキング
- オンラインランキングとローカルランキングを切り替えて表示できるようにする
- 期間は `今日`、`週間`、`月間`、`全期間` で切り替えられるようにする
- オンラインランキングは1万位まで見られるようにし、ページングまたは仮想スクロールで表示する
- 必要なら撃破数などの補助記録
- オンライン取得失敗時のローカルスコア表示
- タイトルへ戻るボタン

スイカゲーム準拠の表示仕様:

- 画面上部にスコアボード見出しを置く
- 自己ベストをランキング一覧より上に大きく表示する
- 自己ベストの下に `オンライン / ローカル` の切り替えと期間切り替えを置く
- ランキングは順位、プレイヤー名、距離スコアを1行にまとめた縦リストにする
- 1位から3位はメダル、王冠、色分けなどで強調し、4位以降は通常行にする
- スコアは右寄せで揃え、単位は `m` にする
- 撃破数などの補助記録は主表示に混ぜず、必要な場合のみ小さなサブ情報として扱う
- オンライン取得成功時も失敗時のローカル表示時も、同じレイアウトを維持する

### 10.5 設定画面

- プレイヤー名
- 音量
- 効果音オン・オフ
- BGMオン・オフ
- タイトルへ戻るボタン

プレイヤー名は設定画面で変更する。リザルト画面では名前入力を出さず、現在設定されている名前でスコアを扱う。

## 11. アセット一覧

### 11.1 主人公

- 子供の竜・飛行アニメ
  - 羽ばたき上
  - 羽ばたき中
  - 羽ばたき下
  - 戻り中間
  - 予備フレーム
- 子供の竜・ダメージまたは衝突表情

羽ばたきは滑らか重視のため、5から8フレームを目標にする。

ゲーム内表示サイズは画面短辺の約14%から15%を基準にする。横位置は画面左寄りに固定し、フラッピーバードの鳥と同じく「小さすぎず、上下操作の結果がはっきり見える」サイズにする。

### 11.2 攻撃

- 炎弾または炎ビーム
- 命中エフェクト
- 敵撃破エフェクト

炎弾は対象敵へ向かう角度に合わせて描画時に回転する。アセットの元向きが固定でも、プレイ画面では弾の進行方向と一致させる。

### 11.3 敵

- コウモリ
- ツノバード
- ぷかぷかスライム
- 空中メイジ
- アーマーガーゴイル
- 敵弾

各敵は最低限、通常状態と撃破エフェクト用の見た目を用意する。

ゲーム内表示サイズは小型敵を画面短辺の約13%、中型敵を約16%、大型敵を約20%にする。敵の並びやゲート状配置は、キャラクター本体の大きさと通路幅をセットで調整し、フラッピーバードの土管のように一目で危険範囲が読める見た目を優先する。

右から左へ進む敵は、見た目も左向きになるように描画時の左右反転で補正する。

### 11.5 背景

背景は新規生成せず、既存の ActionMaker-Steam 背景素材を利用する。

素材元:

- `D:\Documents\ActionMaker-Steam\assets\backgrounds`

標準背景:

- `sky.png` を初期プレイ背景にする

差し替え候補:

| ファイル | サイズ | 用途案 |
| --- | --- | --- |
| `sky.png` | 1774 x 887 | 標準背景 |
| `sunset.png` | 1983 x 793 | 夕方テーマ |
| `space.png` | 2172 x 724 | 高難度・宇宙テーマ |
| `snow.png` | 2172 x 724 | 雪空テーマ |
| `lava.png` | 2172 x 724 | 危険エリア風テーマ |
| `meadow.png` | 2172 x 724 | 明るい草原上空テーマ |
| `forest.png` | 1983 x 793 | 森の上空テーマ |
| `cavern.png` | 1983 x 793 | 洞窟テーマ |

縦画面での扱い:

- 元画像は横長なので、スマホ縦画面ではアスペクト比を保ったまま画面の高さに合わせて拡大・縮小する
- 高さ合わせ後の画像幅が画面幅を超える部分を、横スクロール用の余白として使う
- ドラゴンが進んでいる感覚を出すため、背景は右から左へ少しずつスクロールする
- 背景スクロール速度は敵や地形より遅くする。初期値はゲーム速度の 15% から 25% 程度を目安にする
- 横方向の端まで到達したら、同じ画像をループ接続して継続スクロールする
- ループのつなぎ目処理は ActionMaker-Steam の `tools/make_seamless.py` を参考にする
- 具体的には、背景を半幅シフトして継ぎ目を中央に置き、継ぎ目左右の色差を smoothstep のフェード幅で補正し、最後に継ぎ目周辺の細い帯へ水平ぼかしをかける方式を採用する
- `tools/make_seamless.py` は入力画像を上書きするため、元プロジェクトの背景画像には直接実行しない。必ずこのゲーム用にコピーした背景ファイルへ適用する
- それでもつなぎ目が目立つ背景だけ、左右反転した2枚目をつなげる方式をフォールバックにする
- 画面上下に空白を出さない
- プレイヤー、敵、弾の視認性が落ちる背景は暗さ・彩度・ぼかしを調整する
- 実装時は元プロジェクトを直接変更せず、このゲーム用の `assets/backgrounds` に必要ファイルをコピーして参照する

## 12. 画像生成方針

アセットは `imagegen` スキルで生成する。

### 12.1 共通プロンプト方針

- 2D hand-drawn mobile game sprite
- cute fantasy style
- clean silhouette
- readable at small size
- transparent background用に、生成時は単色クロマキー背景を使い、後処理で透過する
- no text
- no watermark
- consistent lighting and line weight

### 12.2 主人公生成方針

ぽっちゃりした子供の竜を横向きで描く。スマホゲームのプレイヤーキャラクターとして小さく表示しても読みやすいシルエットにする。羽ばたきアニメ用に、同じ体型・同じ向き・同じ表情で翼位置だけを変えた複数フレームを生成する。
